This is the 9-chapter initial version of the wsbk project index.html file, tested within IE6.0 and Mozilla 1.4. We place it in a Tomcat webapps directory, but the early examples may need to be started as file URIs, not via "http://localhost..." in order to bypass security constraints on downloaded pages. (This depends on your security configuration, of course.) Beginning with Chapter 4, running the code requires a database which by default is a MySQL database called wsbkdb; you must install MySQL or edit our XML configuration files, changing "jdbc:mysql://localhost/wsbkdb" as appropriate, and possibly editting the SQL commands in those configuration files. The code of Chapter 4 runs on Windows by clicking our soapUtil/DBService.bat; you can initialize data tables within the browser client. Chapters 5 and 6 use soapUtil/DBAuthService.bat similarly. Chapter 6 also requires an internet connection for examples which reach Amazon's web site or http://www.n-topus.com, and it further requires soapUtil/DBAuthService2.bat for its final example. Chapter 9 is based on command-line Java invocations, each of which is shown for Windows users in a .bat file; Linux users should have no trouble adapting these.

xmlhttpFrameTop google demonstration just shows the elements of a Google SOAP call/result, with all three calls demonstrated and their XML input and output placed in frames.

googleShare uses the search call to find a "mindShare", or "googleShare", value. This is an idea created by Steven Berlin Johnson and implemented in many ways and many languages. The idea is simply that if you count the number of occurrences of "relativity" on the web with Google, and then count the number of occurrences of "Einstein relativity" (i.e., pages where both terms occur), you have a crude measure of how far the discussion of relativity is "penetrated" by discussion, or at least mention, of Einstein. This is Einstein's mindshare of relativity. If relativity is never mentioned without Einstein, then the googleShare value will be 100%; if Einstein is never mentioned in the context of relativity, the googleShare value will be 0%.

Exactly the same code can then be used to get a sense of how specialized the "Einstein" term is to relativity: in effect, relativity's mindshare of Einstein. Some will find this obvious, others will find that it is a very strange but useful contortion: just swap the "who" and "what".

The base64 demo shows the retrieval of a page from Google's cache. These pages are returned in base64 encoding, which is briefly explained and illustrated at base64

The weather/xmlhttpWeatherTop.html demo shows weather reporting using the same Javascript technique as we used for Google, very slightly generalized. (At the moment, weather reporting is available for some codes for some airports. For example, looking up "Atlanta" finds various possible codes; "KPDK" works.) The amazon/xmlhttpAmaKeywordTop.html demo does it for Amazon's keyword search, again in the same way, while amazon/xmlhttpAmazonMultiTop.html puts a large group of Amazon services onto a page. As another kind of extension, we show in amazon/xmlhttpAmaKeyTableTop.html how to put the output into a table of cover-page images, links, and straight data extracted from the XML.

Having generalized the Javascript, we produce Chapter 3's Java applet-based examples, beginning with the somewhat-specialized amazon/appletAmaKeyword1Top.html which generalizes easily to amazon/appletAmaKeywordTop.html The weather/appletWeatherTop.html demo shows weather reporting using the same technique, and amazon/multiserviceAppletTop.html combines three services on the same page, using the same applet.
